Lightweight, YAML-Driven Tools for API Testing & Mocking

The Perfect API Development Companion

Two complementary tools that work together to streamline your API development and testing workflow:

ramjam tests your APIs with declarative YAML workflows, while jimjam provides configurable mock API responses. Together, they eliminate the need for complex test frameworks or heavyweight mocking solutions.

ramjam

Go + Cobra

A CLI tool for executing HTTP API workflows defined in YAML files.

ramjam lets you define entire API test scenarios declaratively, with support for variable substitution, response validation, and value capturing across requests.

Key Features

  • YAML-based workflow definitions
  • Variable substitution and capture
  • JSONPath response validation
  • Authentication flow support
  • Multi-step API workflows
  • Zero code required
  • Support for calling gRPC endpoints (coming soon)

Quick Start

# Install
go install github.com/michaelmccabe/ramjam@latest

# Run a workflow
ramjam run workflow.yaml

# Run all tests in a folder
ramjam run ./tests/integration/

Perfect For

Integration testing, API validation, CI/CD pipelines, authentication flows, and multi-step test scenarios.

jimjam

Rust + Axum

A configurable HTTP mock server that serves responses based on YAML-defined rules.

jimjam provides flexible API mocking with conditional responses, path parameters, and file-based configurations. Perfect for development, testing, and demos.

Key Features

  • YAML-based mock definitions
  • Conditional response matching
  • Dynamic path parameters
  • Configurable response delays
  • File-based response bodies
  • Hot-reload ready architecture (coming soon)
  • Support for gRPC endpoints (coming soon)

Quick Start

 # Build
cargo build --release

# Run
cargo run

# Server starts on
http://127.0.0.1:8080

Perfect For

Development environments, frontend development without backend, API demos, testing edge cases, and simulating slow/failing services.

Better Together

Use jimjam to mock your APIs during development, then use ramjam to test them in integration.

Both tools emphasize simplicity, declarative configuration, and zero-code workflows.
